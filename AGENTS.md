# AGENTS.md

Guidance for AI coding agents working on this repo.

## What this site is

Static portfolio site for an interior design studio (Cecilia Pozzi). Originally a WordPress site, mirrored with `wget`, then migrated to Eleventy. Deployed to GitHub Pages.

## Source of truth

- **`src/`** — Eleventy source. All edits go here.
- **`legacy/`** — The original WordPress static mirror. Never modify. Use only to recover content, image filenames, or structural references that were lost during the migration. Not part of any build.
- **`_site/`** — Build output. Never edit directly; it is regenerated by `npx @11ty/eleventy`.
- **`node_modules/`** — Dependencies. Not committed.

## How the build works

`npm run build` runs Eleventy with `src/` as input and emits `_site/`. Pagination over `_data/projects.json` generates one page per project at `/projects/<slug>/`. Other pages (`about`, `contact`, `furniture/*`, `legal/*`) are individual `.njk` files with their own front matter.

The `HtmlBasePlugin` rewrites root-absolute URLs (`/assets/...`) to include `PATH_PREFIX` at build time. In CI, `actions/configure-pages` sets `PATH_PREFIX` to the GitHub Pages base path. Locally it defaults to `/`.

## Editing rules

### Use the `| url` filter for site-internal URLs in new templates

Within Nunjucks templates, always pipe absolute internal paths through Eleventy's `url` filter so they pick up `PATH_PREFIX`:

```njk
<a href="{{ '/projects/' | url }}">Projects</a>
<link rel="stylesheet" href="{{ '/assets/css/styles.css' | url }}">
```

The `HtmlBasePlugin` also rewrites raw `/...` paths in attributes, so existing extracted markup works without `| url`. Prefer `| url` in new code for clarity.

### Body classes are required

The CSS bundle scopes many selectors to WordPress-era body classes (`.wp-theme-calacatta`, `.wpb-js-composer`, `.home`, `.single-project`, etc.). Without them, fonts, hero sections, and the cookie banner render unstyled. The base layout ([src/_includes/layouts/base.njk](src/_includes/layouts/base.njk)) always injects `wp-custom-logo wp-theme-calacatta wpb-js-composer js-comp-ver-8.6.1 vc_responsive`, plus the page-specific `bodyClass` from front matter.

When adding a new page, set `bodyClass` in its front matter (mirror an adjacent page, e.g. `wp-singular page-template-default page <slug>`).

### Image conventions

- Each project's images live in `src/assets/img/projects/<slug>/`.
- For every JPG/PNG include the WebP variant beside it (`foo.jpg` + `foo.jpg.webp`). Templates render `<picture>` with the WebP `<source>` first.
- Always include `width` and `height` attributes on `<img>` to avoid Cumulative Layout Shift.
- Non-hero images use `loading="lazy"` (native). Hero gets `fetchpriority="high"`.
- Alt text for project images follows: `"{title}, {location} — image N of M"` (see [src/_includes/layouts/project.njk](src/_includes/layouts/project.njk)).

### JSON-LD

- Site-wide `Organization` + `WebSite` is in [src/_includes/partials/head.njk](src/_includes/partials/head.njk) and renders on every page.
- Project pages also emit `CreativeWork` and `BreadcrumbList` via [src/_includes/layouts/project.njk](src/_includes/layouts/project.njk).
- When adding structured data, validate with Google's Rich Results Test before merging.

## Common tasks

### Add a new project

See [README.md](README.md). The data lives in `src/_data/projects.json`; no template changes needed.

### Change site metadata

`src/_data/site.json`. Used by JSON-LD, footer, page titles, OG tags. Note: `site.url` is the production deploy URL; used to build absolute URLs in JSON-LD and `og:image`.

### Change navigation

`src/_includes/partials/header.njk` and `footer.njk` are static. Edit hrefs there.

### Disable Google Analytics

Remove or empty `gtmId` in `src/_data/site.json`. The GTM `<script>` in [src/_includes/partials/head.njk](src/_includes/partials/head.njk) is wrapped in `{% if site.gtmId %}`.

## What NOT to do

- **Do not rewrite the CSS bundle from scratch.** `src/assets/css/styles.css` is a 480 KB compiled bundle inherited from the original WordPress install (Litespeed + Yoast + Complianz + theme + Visual Composer). It is opaque but works. Replacing it without rebuilding the theme from scratch will break the site. PurgeCSS would be the right tool if reducing its size becomes a priority.
- **Do not move files out of `src/assets/img/`** without updating `src/_data/projects.json` and the extracted HTML in static `.njk` pages.
- **Do not edit anything inside `legacy/`.** It is the historical reference. If something is missing in `src/`, you may copy from `legacy/` into `src/` but never modify `legacy/` itself.
- **Do not introduce `/wp-content/` paths.** They were intentionally removed during the migration. The site has zero remaining references to `wp-content`, including external ones.
- **Do not change slugs of existing projects** in `_data/projects.json` without also moving their image folders and verifying no inbound links break.

## Verifying changes

```bash
npm run build
# Or, for live preview:
npm run serve
```

Then check:

1. Routes return 200:
   ```bash
   curl -sI http://localhost:8080/ | head -1
   curl -sI http://localhost:8080/projects/ganduxer/ | head -1
   ```
2. No broken image references on the home or any project page (use browser DevTools Network tab).
3. JSON-LD parses (use [validator.schema.org](https://validator.schema.org/) or Google's Rich Results Test on the deployed URL).

## Deployment

CI: [.github/workflows/deploy.yml](.github/workflows/deploy.yml) runs on push to `main` or `eleventy`. The workflow uses `actions/configure-pages@v5` to set `PATH_PREFIX` automatically. Output is uploaded as a Pages artifact and deployed via `actions/deploy-pages@v4`.

Enable Pages in the repo settings: **Settings → Pages → Source: GitHub Actions**.

## Migration scripts

Several Python scripts under `/tmp/` were used for the one-shot migration from the WordPress mirror to Eleventy (data extraction, image reorganization, page conversion). They are not part of the live codebase; do not re-run them. If you need to re-import content from `legacy/`, write a new script and document it here.
